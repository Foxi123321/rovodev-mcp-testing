# Rovo Dev CLI Release Notes v0.11.5

## Release Metadata

- **Version**: 0.11.5
- **Previous Version**: 0.10.2
- **Release Date**: August 19, 2025
- **Total Commits**: 111 changes
- **Repository**: ACRA Python Monorepo

## üöÄ New Features

- **MCP Management UI**: Complete MCP command and management interface with server/tool details panels
- **IDE Integration**: New `/ide` command to install VS Code plugin automatically
- **Loom Video Support**: Added tool to fetch Loom video transcript data for enhanced documentation
- **Session Management**: Enable resuming sessions in generate mode for improved workflow continuity
- **Interactive Commands**: Support for long-running and interactive bash/powershell commands with background execution
- **File Watcher**: Added file watcher functionality for real-time code monitoring
- **Web GUI**: Comprehensive web-based GUI for serve mode with production assets
- **MLflow Integration**: Added MLflow integration for experiment tracking and analytics
- **Project-level MCP Configs**: Support for project-specific MCP server configurations
- **Vi-mode Support**: Added vi-mode text editing capabilities
- **Sandbox Mode**: Enable sandbox for internal dogfooding and testing
- **Git AI Statistics**: Enhanced git AI statistics functionality
- **Comment Verification**: Added CodeNemoCommentVerifierAgent for code review assistance
- **Retry Model**: Improved reliability with retry mechanisms for failed operations

## üêõ Bug Fixes

- **Session Corruption**: Fixed session corruption and infinite recursion issues
- **Jira Integration**: Updated regex for get-jira-issue tool and added backward compatibility for deprecated fields
- **Windows Compatibility**: Multiple fixes for Windows test failures and package compatibility
- **macOS Build**: Resolved macOS build compatibility issues
- **Markdown Truncation**: Fixed markdown truncation in get-confluence-page tool
- **Stream Events**: Removed hallucinated close events and improved event handling
- **Model Context**: Fixed model context limits handling and GPT-5 usage
- **Resource Errors**: Resolved anyio closed resource errors occurring frequently
- **Token Display**: Reverted token display to model ID for consistency
- **Live Display**: Fixed "only one live display may be active at once" error

## üîß Engineering Improvements

- **Dependency Management**: Switched to mlflow-skinny and pruned transitive dependencies
- **CI/CD Enhancements**: Added Windows CI support and web-gui to CI pipeline
- **Code Quality**: Extensive linting fixes and code formatting improvements
- **Documentation**: Added detailed serve mode documentation and auto-update CI for docs
- **Testing**: Improved test structure, removed debug tests, and enhanced package testing
- **Performance**: Optimized warning/error event handling and session growth prevention
- **Library Updates**: Upgraded to pydantic-ai 0.4.3 and pinned rich library version
- **Build Process**: Enhanced production assets build process and packaging improvements
- **Authentication**: Improved auth header encoding for Atlassian MCP integration
