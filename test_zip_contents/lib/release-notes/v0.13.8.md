# Release Notes: v0.13.8

**Release Date:** November 27, 2024  
**Previous Version:** v0.13.5

## Summary

This release includes significant new features, security improvements, and performance optimizations. Key highlights include the introduction of interactive user questions, enhanced authentication security, and improved user experience with version display and interrupt handling.

## New Features

- **Interactive User Questions**: Added support for `ask_user_questions` as a deferred tool, enabling agents to request user input during execution
- **Version Display**: Added version information to the startup banner for better visibility
- **Enhanced Interrupt Handling**: Improved Ctrl+C interrupt handling with graceful cancellation capabilities
- **Environment-based Configuration**: Added environment-based Statsig API key management with tier support
- **Universal Docker Support**: Added system prompt support for universal docker images

## Bug Fixes

- **Security**: Fixed RCE vulnerability by adding session token authentication to serve endpoints
- **Session Management**: Fixed session ID consistency issues in serve mode
- **Tool Documentation**: Corrected parameter naming in grep tool documentation
- **Docker Integration**: Fixed dataset name comparison for prebuilt docker images
- **Interactive Mode**: Improved interactive mode logic separation for better testability
- **Windows Compatibility**: Resolved Windows console errors in prompt toolkit mocking

## Engineering Improvements

- **Performance**: Optimized workspace-specific session loading for better performance
- **Dependencies**: Upgraded Pydantic AI from v1.12.0 to v1.22.0 with API compatibility updates
- **Configuration**: Deprecated `sessions.autoRestore` config field in favor of `--restore` flag
- **Authentication**: Added groups flag for slauth when using autofix-service-evaluation

## Breaking Changes

- **Config Deprecation**: The `sessions.autoRestore` configuration field is now deprecated. Users should use the `--restore` flag instead.
